# âœ… **1. What is Angular Router?**

* Angular Router is a **module that enables navigation** between different views (components) in a single-page application (SPA).
* It allows **URL-based navigation**, so users can bookmark or share links without reloading the page.

**Key points:**

* Uses `<router-outlet>` as a **placeholder** for rendering components
* Supports **lazy loading**, **guards**, **nested routes**, **parameters**

---

# âœ… **2. How Routing Works in Angular**

1. **Import RouterModule** in your module:

```ts
import { RouterModule, Routes } from '@angular/router';

const routes: Routes = [
  { path: '', component: HomeComponent },
  { path: 'about', component: AboutComponent },
  { path: '**', component: PageNotFoundComponent } // fallback
];

@NgModule({
  imports: [RouterModule.forRoot(routes)],
  exports: [RouterModule]
})
export class AppRoutingModule {}
```

---

2. **Add `<router-outlet>` in App Component**

```html
<router-outlet></router-outlet>
```

* Acts as a **dynamic placeholder** for the routed components.

---

3. **Navigate between routes**

* Using **RouterLink** in templates:

```html
<a routerLink="/about">About</a>
```

* Or **programmatically** in TypeScript:

```ts
constructor(private router: Router) {}
goToAbout() {
  this.router.navigate(['/about']);
}
```

---

4. **Optional: Route Parameters**

```ts
{ path: 'product/:id', component: ProductComponent }
```

Access in component:

```ts
constructor(private route: ActivatedRoute) {
  const id = this.route.snapshot.paramMap.get('id');
}
```

# ðŸ”¹ **How Routing Works Internally**

1. User clicks a link or navigates via `router.navigate()`
2. Angular Router **matches the URL** with the defined `Routes`
3. Router **loads the corresponding component** in `<router-outlet>`
4. Lifecycle hooks like `ngOnInit` of the routed component run
5. Optional **guards, resolvers, lazy loading** are triggered before rendering
